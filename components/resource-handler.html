ResourceHandler

ResourceHandler::getLocalStorage()#

Returns all the available local storage mediums (local, s3).

It might be a bit confusing but here getLocalStorage() doesn't mean "storage on the same server as phpReel". It simply means that you store the content yourself and have access to the actual stored content.

  • Returns array
/app/Helpers/Resource/ResourceHandler.php
use App\Helpers\Resource\ResourceHandler;
...
//This will return ['local', 's3']ResourceHandler::getLocalStorage();

ResourceHandler::getExternalStorage()#

Returns all the available external storage mediums (youtube, vimeo).

  • Returns array
/app/Helpers/Resource/ResourceHandler.php
use App\Helpers\Resource\ResourceHandler;
...
//This will return ['youtube', 'vimeo']ResourceHandler::getExternalStorage();

ResourceHandler::checkPremium($fileName, $storage)#

Returns true if video is premium, false if not, null if it doesn't exist.

A video is considered to be premium if you need an active subscription to watch it.

  • $fileName Required string, name of the stored file (different from the name the file it had when you uploaded it)
  • $storage Required string, storage type used when the file was stored (local, s3)
  • Returns bool
/app/Helpers/Resource/ResourceHandler.php
use App\Helpers\Resource\ResourceHandler;
...
ResourceHandler::checkPremium('video.mp4', 's3');

ResourceHandler::addImage($fileName)#

Add a image resource to the database. Return the id of the new row from the image table.

This method doesn't save the actual image, it's used to register the uploaded image to the database so you can keep track of it.

  • $fileName Required string, name of the stored image (different from the name the file it had when you uploaded it)
  • Returns int
/app/Helpers/Resource/ResourceHandler.php
use App\Helpers\Resource\ResourceHandler;
...
ResourceHandler::addImage('image.jpeg');

ResourceHandler::addVideo($fileName, $storage, $premium = 1, $auth = 1)#

Add a video resource to the database. Return the id of the new row from the video table.

This method doesn't save the actual video, it's used to register the uploaded video to the database so you can keep track of it.

  • $fileName Required string, name of the stored video (different from the name the file it had when you uploaded it)
  • $storage Required string, storage type used to store the video (local, s3, youtube, vimeo)
  • $premium Optional int, by default every video is saved as premium (0=non premium, 1=premium)
  • $auth Optional int, by default every video requires auth (0=no auth required, 1=requires auth)
  • Returns int

$premium tells phpReel if a certain video requires an active subscription to be streamed while $auth tells if the user must be logged in or not. You can not set a video that is both $premium = 1 and $auth = 0.

/app/Helpers/Resource/ResourceHandler.php
use App\Helpers\Resource\ResourceHandler;
...
ResourceHandler::addVideo('video.mp4', 's3');

ResourceHandler::delete($fileName, $storage)#

Deletes a specific resource (video, image).

  • $fileName Required string, name of the stored resource to be deleted (different from the name the file it had when you uploaded it)
  • $storage Required string, storage type used to store the video (local, s3)
  • Returns Illuminate\Support\Facades\Storage
/app/Helpers/Resource/ResourceHandler.php
use App\Helpers\Resource\ResourceHandler;
...
ResourceHandler::delete('video.mp4', 's3');

ResourceHandler::updateVideo($fileName, $videoId, $storage, $premium = 1, $auth = 1)#

Update a video resource from the database.

This method doesn't update the actual video, it's used to register the uploaded video to the database so you can keep track of it.

  • $fileName Required string, name of the new stored video (different from the name the file it had when you uploaded it)
  • $videoId Required int, id of the database row that contains that specific video to be updated
  • $storage Required string, new storage type used to store the video (local, s3, youtube, vimeo)
  • $premium Optional int, by default every video is saved as premium (0=non premium, 1=premium)
  • $auth Optional int, by default every video requires auth (0=no auth required, 1=requires auth)

$premium tells phpReel if a certain video requires an active subscription to be streamed while $auth tells if the user must be logged in or not. You can not set a video that is both $premium = 1 and $auth = 0.

/app/Helpers/Resource/ResourceHandler.php
use App\Helpers\Resource\ResourceHandler;
...
ResourceHandler::updateVideo('video.mp4', 5, 'local', 0, 0);

ResourceHandler::updateImage($fileName, $imageId, $storage)#

Update a image resource from the database.

This method doesn't update the actual image, it's used to register the uploaded image to the database so you can keep track of it.

  • $fileName Required string, name of the new stored image (different from the name the file it had when you uploaded it)
  • $imageId Required int, id of the database row that contains that specific image to be updated
  • $storage Required string, new storage type used to store the image (local, s3)
/app/Helpers/Resource/ResourceHandler.php
