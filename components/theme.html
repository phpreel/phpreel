Theme

Theme::existsChildView($view)#

Returns true if a child view exists and false if there is not a child view.

  • $view Required string, path to the view to be checked

  • Returns bool

/app/Helpers/Theme/Theme.php
use Theme;
...
//Checks if the show.blade.php view exists in the child theme or notTheme::existsChildView('show');

Theme::getActiveTheme()#

Checks if there is a child theme defined, if not then it returns the active theme, else it's child theme.

  • Returns string
/app/Helpers/Theme/Theme.php
use Theme;
...
Theme::getActiveTheme();

Theme::getFilePath($path)#

Returns the file path of a file either from the theme or child theme. This should be used for anything that is not a view.

  • $path Required string, path to the file to be checked (starting from theme root)
  • Returns string
/app/Helpers/Theme/Theme.php
use Theme;
...
Theme::getFilePath("css/style.css");

Theme::view($path, $data)#

Returns a specific view. If there is a file with the same name in the child theme folder then that file will be returned, else it will return the file from the actual theme directory. Very similar to what view does in Laravel.

  • $path Required string, path to the view to be rendered
  • $data Required array, data to be passed to the view
  • Returns global view helper
/app/Helpers/Theme/Theme.php
use Theme;
...
Theme::view("show", [    'movie' => $movie,    'subscribed' => true]);

Theme::createFiles($directories, $path)#

Generates the theme files and directories (if they do not exist) based on the directories array. This usually is used with Theme::getDefaultThemeDirectories() as shown in the example bellow.

  • $directories Required array, key-value pairs, where the key is the name of the directory and the pairs are arrays of strings representing the files present in those directories. If the array is left empty in means that the directory is empty.
  • $directories Required string, path to where to store the files and directories.

This is how the $directories array might look like:

$directories = [    'auth' => ['forgot-password.blade.php', 'login.blade.php', 'register.blade.php', 'reset-password.blade.php'],     'categories' => ['cast.blade.php', 'genre.blade.php', 'rating.blade.php', 'release.blade.php'],     'css' => [],     'episodes' => ['show.blade.php'],     'home' => ['home.blade.php'],    'error' => ['error.blade.php'],    'img' => [],     'js' => [],     'lang' => ['default' => ['default.json']],     'layouts' => ['layout.blade.php'],     'movie' => ['index.blade.php', 'show.blade.php'],     'pagination' => ['simple-pagination.blade.php'],     'search' => ['index.blade.php'],     'series' => ['index.blade.php', 'show.blade.php'],     'subscribe' => ['index.blade.php'],     'trailer' => ['show.blade.php'],     'user' => ['index.blade.php']];
/app/Helpers/Theme/Theme.php
use Theme;
...
Theme::createFiles(Theme::getDefaultThemeDirectories(), 'themes/default');

Theme::generateConfig($theme, $themeName, $description, $author, $themeUrl, $version, $license, $licenseUrl)#

Generates the config file for a new theme.

  • $theme Required string, name of the directory where the theme will be saved themes\$theme
  • $themeName Required string, name that will be displayed in the dashboard theme section
  • $description Required string, description of the theme
  • $author Required string, author of the theme
  • $themeUrl Required string, URL to a webpage dedicated to the theme
  • $version Required string, current version of the theme
  • $license Required string, license used by the theme (MIT, BSD, ...)
  • $licenseUrl Required string, URL to the license contents
/app/Helpers/Theme/Theme.php
use Theme;
...
Theme::generateConfig('default', 'Default theme', 'Amazing description', 'Paul Balan', 'phpreel.org', '1.0.0', 'MIT', 'https://github.com/phpreel/phpreel/blob/main/LICENSE');

Theme::syncLang()#

Synchronization between the lang file of the theme and the resources/lang file of Laravel. In short, it takes the files from the lang file of the theme and copies them to resources/lang. This is done so the user can update their language preferences.

/app/Helpers/Theme/Theme.php
use Theme;
...
Theme::syncLang());
